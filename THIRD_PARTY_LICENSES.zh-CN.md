# 第三方许可证说明

除非某个文件另有说明，本仓库自身的源码和文档按 Apache License 2.0 提供。

## 源码仓库范围

- 这个公开源码仓库不会把 `node_modules/` 作为受 git 跟踪的源码内容提交。
- 第三方 npm 依赖是在用户安装或构建项目时由包管理器单独下载的。
- 这些第三方依赖仍然分别受其各自许可证约束。

## 本地审计说明

在 2026-03-21，对本仓库已安装 npm 依赖树做的一次本地元数据扫描中，没有发现显式声明为 `GPL`、`AGPL`、`SSPL` 或 `BUSL` 的包许可证。

同一次本地扫描里，确实出现了以下这类宽松或需要保留声明的许可证：

- `MIT`
- `Apache-2.0`
- `BSD-2-Clause`
- `BSD-3-Clause`
- `ISC`
- `MPL-2.0`
- `BlueOak-1.0.0`
- `CC-BY-4.0`
- `CC0-1.0`

有些已安装包在本地元数据里没有暴露出简单的 `license` 字段，尤其是平台相关包装包和本地 workspace 包。因此，这份文件只是面向仓库维护者的信息性审计说明，不能替代针对具体发布产物的法律复核。

## 如果你分发打包产物

如果你要分发容器、二进制发布包、预构建前端资源或任何包含第三方材料的打包产物，请针对最终打包内容重新复核，并按需要更新随附的许可证材料。

至少应重新检查：

- 是否需要随产物一起分发额外的第三方许可证文本
- 是否需要附带额外的归属或声明信息
- 最终产物中的 `LICENSE` 和 `NOTICE` 是否仍与实际打包内容一致
