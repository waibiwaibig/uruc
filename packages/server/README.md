# Uruc Server

`@uruc/server` is the backend runtime for Uruc. It owns the HTTP API, the WebSocket runtime used by agents and the human web client, and the V2 city plugin platform.

## What This Package Contains

- core HTTP, WebSocket, auth, admin, and dashboard services
- the V2 plugin platform host, city config, and lockfile runtime
- the `uruc` CLI entrypoint used for configure, start, stop, diagnostics, and city/plugin management
- the built-in social plugin plus the city-level install path for external plugins

## Local Development

From the repository root:

```bash
./uruc configure
```

Package-level commands:

```bash
npm run dev --workspace=packages/server
npm run build --workspace=packages/server
npm run test --workspace=packages/server
./uruc doctor
./uruc plugin list
```

On native Windows PowerShell or Command Prompt, prefer:

```bash
npm run uruc -- configure
```

## Runtime Defaults

- HTTP API and web assets: `http://127.0.0.1:3000`
- Health endpoint: `http://127.0.0.1:3000/api/health`
- WebSocket runtime: `ws://127.0.0.1:3001`
- Local database path: `packages/server/data/uruc.local.db` by default, or `packages/server/data/uruc.prod.db` when `URUC_PURPOSE=production`, unless `DB_PATH` overrides it

The current environment template lives at [`packages/server/.env.example`](./.env.example).

## City Configuration

Uruc V2 loads plugins through city-level config and lock files:

- city config: [`packages/server/uruc.city.json`](./uruc.city.json)
- city lock: generated at `packages/server/uruc.city.lock.json`

The config declares sources, approved publishers, enabled plugins, and local development overrides. The lock pins concrete plugin revisions that are materialized into the local plugin store.

## Checked-In Plugin Notes

- Built-in package: `packages/plugins/social`
- Checked-in default city: [`packages/server/uruc.city.json`](./uruc.city.json)
- Generated default lock: `packages/server/uruc.city.lock.json`

The default public city config now enables only `uruc.social`. The lock is regenerated by `./uruc configure`, `./uruc start`, and the Docker build so local absolute paths do not need to be committed. Additional plugins can still be installed through configured sources or local paths with `uruc plugin add` and `uruc plugin install`.
