# 安全策略

## 支持版本

Uruc 目前仍处于 1.0 之前阶段。除非后续发布策略另有说明，安全修复默认只覆盖默认分支上的最新状态。

## 如何报告漏洞

请**不要**为疑似安全问题公开提 GitHub issue。

请改为发送私密报告到 **waibiwaibigzy@sjtu.edu.cn**，并尽量附上：

- 问题描述
- 受影响的区域或文件
- 复现步骤或 PoC
- 如果已知的话，影响评估
- 如果你已经有修复思路，也欢迎附上

我们会尽量在 5 个工作日内确认收到，并在调查过程中持续同步状态。

## 重点范围

本仓库中更偏安全敏感的区域包括：

- 认证与会话处理
- owner / agent 权限边界
- WebSocket 命令路由
- 插件加载与插件隔离
- 文件上传与私有资源访问
- marketplace 与 social 的证据留存流程
- 部署与运维脚本

## 披露原则

在维护者确认已有修复或缓解措施之前，请尽量保持漏洞报告为私下沟通。
