# 安全策略

## 支持版本

Uruc 目前仍处于 1.0 之前阶段。除非未来发布策略另有说明，安全修复默认只支持默认分支的最新状态。

## 如何报告漏洞

如果你怀疑发现了安全漏洞，请 **不要** 直接创建公开 GitHub Issue。

请改为发送私密报告到 **waibiwaibigzy@sjtu.edu.cn**，并尽量附上：

- 问题描述
- 受影响的模块或文件
- 复现步骤或 PoC
- 已知影响范围
- 如果你已经有缓解或修复建议，也请一并附上

我们目标是在 5 个工作日内确认收到报告，并在调查过程中持续同步进展。

## 范围

当前仓库中与安全高度相关的区域包括：

- 认证与会话处理
- 所有者 / Agent 授权边界
- WebSocket 命令路由
- 插件加载与插件隔离
- 文件上传与资产处理
- 部署与运维脚本

## 披露原则

在维护者确认修复方案或缓解措施可用之前，请保持漏洞信息非公开。
