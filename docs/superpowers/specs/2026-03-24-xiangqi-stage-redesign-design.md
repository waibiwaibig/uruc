# Xiangqi Stage Redesign Design

**Date:** 2026-03-24

## Goal

在不改动中国象棋插件的房间、观战、Elo、时钟、命令和右侧工作区结构的前提下，重做主舞台的棋盘与棋子表现，使其回到经典中国象棋视觉：木质棋盘、九宫、河界、盘星、红黑圆盘字子，以及与之统一的席位卡、吃子栏和结果浮层。

## Confirmed Facts

- 当前问题不是单一配色问题，而是实现结构错误：
  - `packages/plugins/chinese-chess/frontend/ChineseChessPage.tsx` 渲染的是中国象棋语义类名，如 `chinese-chess-cell`、`chinese-chess-river`、`chinese-chess-files`、`chinese-chess-ranks`
  - `packages/plugins/chinese-chess/frontend/chinese-chess.css` 仍保留了西洋棋式 `8x8` 方格样式和旧类名 `chinese-chess-square`
  - JSX 与 CSS 类名脱节，导致棋盘尺寸、格线和棋子样式全部失真
- 标准中国象棋棋盘是线盘而不是西洋棋方格盘：
  - 9 条纵线、10 条横线
  - 棋子落在线交点
  - 双方各有九宫斜线
  - 中间有河界
  - 常见盘面含炮位与兵位盘星
- 标准中国象棋棋子是红黑圆盘字子，而不是西洋棋立体棋形或双色方格棋子。

## Non-Goals

- 不修改后端规则引擎、命令协议或对局状态字段。
- 不改动右侧工作区的信息架构。
- 不新增动画化拖拽、音效或额外视觉特效。
- 不引入新的共享棋类 UI 框架。

## Architecture

### DOM Structure

保留 `ChineseChessPage.tsx` 现有页面框架：

- 左侧主舞台 `chinese-chess-main-stage`
- 右侧工作区 `chinese-chess-right-rail`
- 房间、记录、历史、榜单等 tab 结构

只重做主舞台内部的棋盘区：

- 棋盘外框改为“棋案/棋桌”容器
- 上下席位卡贴近棋盘上下边
- 棋盘本体使用专用 overlay 绘制线盘
- 70 个交点按钮作为可点击落子点
- 河界、九宫、盘星作为棋盘装饰层的一部分
- 吃子区和结果浮层统一到木盘视觉体系

### Board Rendering

前端继续使用 `positionFen` 解析出的 `10x9` 矩阵，但视觉上不再把每个位置画成方格，而是画成交点：

- 棋盘底板负责木纹、外框、河界文案
- SVG overlay 负责：
  - 9 条纵线
  - 上下半场横线
  - 两个九宫斜线组
  - 盘星
- 交点按钮只负责交互态和棋子承载
- 棋子改为传统圆盘字子的 SVG 资产，字面与子面绑定在同一矢量图内，缩放时一起变化
- 河界文案使用偏古典的楷体/书法字体栈，而不是通用无衬线字体

### Visual Direction

视觉主轴明确为：

- 材质：温润木色棋盘 + 深漆木外框
- 对比：浅木盘面上用深棕线条；红方朱砂、黑方墨色
- 配套区域：
  - 席位卡从冷灰玻璃感改成半透明漆木/宣纸质感
  - 结果浮层从通用暗色卡片改成与棋盘一致的木色浮牌
  - 吃子栏使用更轻的书卷式信息带，而不是独立冷色块

### Layout Priority Refinement

为了让棋盘重新成为页面主角，布局需要进一步收缩外围信息：

- 删除当前棋盘上下方的大型玩家席位卡
- 改为两个极简玩家角标，分别贴在棋盘左上角和左下角边缘
- 角标使用单行胶囊结构，只保留头像缩写、玩家名、时钟
- 不再单独展示阵营文案；状态只能作为极轻提示，不能变成第二行主信息
- 桌面布局从“左侧适配 + 右栏偏大”改为“棋盘优先占据左侧最大可用面，右栏按剩余空间自适应”
- 移动端继续上下堆叠，但棋盘仍优先保持最大宽度，角标继续附着在棋盘左侧上下边缘

## Interaction

- 继续沿用当前点击选中与点击目标点走子的逻辑。
- 当前可走目标点只改变交点按钮和棋子周围提示，不再高亮整块方格。
- 最近一步改为交点环形标记或柔和底光。
- 非交互态预览盘与对局盘共用同一棋盘结构，只切换禁用态和强调态。

## Testing

需要补上最小回归保护，确保之后不会退回西洋棋结构：

- 页面静态渲染测试应断言新的传统棋盘结构类名存在
- CSS 测试应断言不存在 `8x8` 棋盘网格和旧的 `chinese-chess-square` 主渲染依赖
- 页面测试应断言河界文案与棋盘 overlay 标记存在

## Acceptance Criteria

- 页面主棋盘视觉上不再出现西洋棋明暗格。
- 棋盘带有河界、九宫和盘星。
- 棋子为红黑圆盘字子的完整 SVG 资产，并稳定居中在交点。
- 棋盘上下不再出现大号玩家信息卡。
- 棋盘左上角与左下角各有一个单行玩家角标。
- 桌面端棋盘比右侧功能区拥有更高视觉优先级和更大的可用宽度。
- 主舞台、席位卡、吃子栏和结果浮层视觉语言统一。
- 现有房间/观战/走子流程不回归。
