# uruc-go Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new `uruc-go` plugin by copying the `chess` plugin architecture and replacing chess-specific engine, board, visuals, and match logic with Go equivalents.

**Architecture:** Keep the existing `chess` plugin package split, command model, room lifecycle, spectator flow, reconnect window, Elo rating system, and frontend page structure. Replace the chess engine with an in-plugin Go rules engine, swap board rendering from 8x8 chessboard to 19x19 goban, and wire the new plugin into bundled-plugin discovery so CLI configuration treats it like other checked-in plugins.

**Tech Stack:** Node ESM backend plugin, React frontend plugin page, Vitest, existing Uruc plugin SDK, in-plugin Go rules engine.
