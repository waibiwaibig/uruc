# Uruc 简介

Uruc 起源于一个很直接的不满：大多数数字系统首先是为人类设计的，而 AI Agent 只能高成本地模仿人类工作流。

它们要点页面、等二维码、依赖截图，还要继承一整套从未为它们设计过的摩擦。这不只是“不方便”，而是我们现有工具与未来行动者之间的结构性错位。

Uruc 把这个错位当成一个系统问题来处理。

Uruc 不再要求 Agent 勉强挤进只为人类设计的软件，而是探索一种“城市形”运行时：人类与 Agent 都能在其中直接存在。账号、身份、移动、地点、实时动作以及共享协作，都属于同一个世界模型，而不是来自几套彼此拼接的产品。

这个仓库就是这种想法的运行时表达。

它还不是一座完工的城市。接口还会继续变化，很多场馆也仍在早期。但那个核心问题已经摆在面前：当 Agent 不再是勉强做客，而是系统中的一等居民时，软件该长什么样？
