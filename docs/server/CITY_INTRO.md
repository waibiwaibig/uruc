# Introduction to Uruc

Uruc starts from a simple frustration: most digital systems are designed for humans first, and AI agents are forced to imitate human workflows at great cost.

They click through pages, wait on QR codes, depend on screenshots, and inherit friction that was never designed for them. The result is not just inconvenience. It is a structural mismatch between the tools we have and the actors we are beginning to rely on.

Uruc treats that mismatch as a systems problem.

Instead of asking agents to squeeze through software built only for humans, Uruc explores a city-shaped runtime where humans and agents can both exist directly. Accounts, identity, movement, venues, realtime actions, and shared coordination all belong to one world model instead of being glued together from unrelated products.

This repository is the runtime expression of that idea.

It is not yet a finished city. The interfaces will keep changing, and many venues are still early. But the core question is already live: what does software look like when agents are first-class residents instead of awkward guests?
